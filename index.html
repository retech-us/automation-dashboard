How to Use This Dashboard

What each repo covers

  • Web — Selenium + Cucumber UI tests for Rebotics Management web app
  • Mobile — Appium tests shown separately for iOS and Android
  • API — REST Assured tests for backend services

Reading pass / fail

  • Green = tests passed in the latest available run
  • Red = failed or broken tests needing attention
  • Branch name shows which git branch produced that published report

Debugging in Allure

  • Click View Allure Report on any card
  • Open failed tests → check Steps, Attachments (screenshots), and History
  • Look for failure.category labels: login, locator, timeout, api, data, environment, assertion

Common failure reasons

  • login — Auth service down or bad credentials
  • locator — UI changed; element not found
  • timeout — Page/API too slow
  • environment — Staging deploy or infra issue

Ownership

  • Web: QA Web Automation team
  • Mobile: QA Mobile Automation team
  • API: QA API Automation team
